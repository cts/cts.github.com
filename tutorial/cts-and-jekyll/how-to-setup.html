Setting up CTS

Now that you've gotten Jekyll all installed, let's start setting up CTS.

Open up your terminal, go to the directory where you have your Jekyll blog or you want to set one up, and run:

npm install cts

Now you'll be able run CTS commands from the terminal. If you run just cts you should see a help screen.



Initializing Your Blog

We need to set up some files and folders now --this is different based on if your directory is already a Jekyll blog directory or not (if it's not, it should be empty!).

If you've already got a Jekyll environment in this directory:

You're going to run:

cts setup jekyll

Don't worry about your files! We won't overwrite any of them. If we're adding a file with the same name (we probably will), we'll just rename yours to whatever-backup.ext. This command is going to add some files to the root directory, and to the _includes, _layouts, and _plugins folders. It'll also change your _config.yml file, but just by one line: check the bottom of the file, and it'll say theme: mog.

If you're starting new and fresh:

You're going to run:

cts setup jekyll --new

This command is going to start setting up the directory structure for you. It's going to look a lot like the directory structure that the Jekyll getting started guide showed you: a _layouts, _includes, and _plugins folder, some dummy content in the _posts folder, a place to store images, an index.html, a few CTS files, and a _config.yml file.

You have to have a _config.yml file if you want this to be a Jekyll environment! Read more about it here. Right now there's just one line of code in it: theme: mog. Later you can add new properties, but this is all you need right now.


Managing Your Files

All your files are in place now. Let's talk through what was added. Run your mouse over a file or folder for more specific information about it, or run your mouse over the blurbs to see what files we're talking about.
Files that are dummy content, made for new Jekyll environments, are not shown in this view.

  • _includes
    • default.html
  • _layouts
    • list.html
    • page.html
    • post.html
  • _plugins
    • _list_gen.rb
  • mockups
    • mog
      • default.html
      • index.html
      • list.html
      • mog.cts
      • page.html
      • post.html
  • _config.yml
  • default.cts
  • index.cts
  • index.html
  • list.cts
  • page.cts
  • post.cts
  • There are four different types of pages: index, list, page, and post.
  • Each of these pages uses {% raw %}{% include default.html %}{% endraw %}. default.html is where all of their common content, like the title of the blog, navbar contents, and short lists of recent posts, archives, and categories, resides.
  • The only thing that these HTML documents contain is content! If you were to look at this html alone, it would just be gross lists and spans. If you ever want to edit content, these files are where you do it. Most of what you'll want to edit is in default.html.
  • All of the actual styling is in the mockups folder. mockups holds all of the themes you have installed.
  • Right now, the only theme you have installed is Mog. It was adapted from the theme on Wordpress. We'll show you how to install more themes later!
  • These four html files are the counterparts of the four in the root directory and in _layouts. This is the structure that's going to hold your content. If you look in them, you'll see absolutely no content, but a lot of divs!
  • Just like in the main html files, the HTML files for the theme also have some stuff in common (in this case, structures, not content!). That's all stored in default.html -the counterpart of the default.html in the root directory.
  • Now for some CTS! Each of the four types of pages has its own CTS file: index.cts, list.cts, page.cts, and post.cts. These files will take the content from the main pages and put it in the themed html files.
  • The default.cts file serves the same purpose as the default.html file, only for CTS: the main pages share a lot of the same CTS rules (just like they share a lot of the same HTML elements), so all of the CTS the pages have in common goes in here!
  • The last CTS file is mog.cts. Each of the themes have their own specific HTML files, and their own specific CTS file. The CTS file does two things: first, it loads the theme's CSS and Javascript files from Wordpress; second, it puts everything from the theme's default.html file into the theme's main pages. It's like the {% raw %}{% include default.html %}{% endraw %} in the main pages, but with CTS --no Liquid required!
  • We've provided a plugin for you that will make switching over from Wordpress a little smoother. This plugin is a generator that will create pages that list your posts by tag, category, and month written.
